# AdventureX 申请叙事与首页互动增强设计

日期：2026-07-11

## 1. 目标

为 AdventureX 2026 形成一套互相印证的申请材料：报名表解释陈思翰为什么持续构建、如何判断问题、怎样与人协作；个人主页展示真实项目、构建过程与投入证据；两页简历提供便于快速核验的正式经历。

核心定位保持为“跨医学、科研与产品的独立构建者”。本轮不重做网站信息架构和配色，而是在现有 A 方向上加强首页互动、产品思维、构建激情与可核验投入。

## 2. AdventureX 审核信号

官方公开审核原则强调综合评估，重点是申请者能否在活动中与别人一起做出有意思的东西，并关注其做过什么、如何学习、为什么参加，以及是否真的想动手构建。因此申请材料按以下优先级组织：

1. 真实问题与个人观察，而非抽象赛道口号。
2. 从问题到可运行产品的完整行动链。
3. 面对陌生领域、技术分歧和时间压力时的协作方式。
4. 可验证的学习速度、交付记录和持续投入。
5. 个人趣味、好奇心和不模板化的表达。

## 3. 统一叙事

主线不是“医学生也会写代码”，而是：在免疫学与单细胞研究中反复遇到工具断层和流程不可复用的问题，于是开始把科研判断写进工作流、Agent、质量门和产品界面，并逐步将这种构建能力用于观点调研、时间管理、内容运营和创业协作。

各类证据承担不同角色：

- `scrna-omics`：科研理解、系统设计与 AI4S 产品判断的核心代表作。
- 镜子：复杂调研流程、多智能体协作、质量控制与可交付报告。
- TimeBox：从个人问题出发持续迭代产品，并处理记忆、工具调用和 AI 交互。
- GoGoWork：团队项目中的产品协作、供需匹配与履约流程理解。
- 小红书与微信社群：公开表达、用户反馈、内容分发和持续运营。
- 小产品索引：快速试错与把具体问题做成工具的密度。

## 4. 首页互动设计：Builder in Motion

### 4.1 首屏

- 保留全屏研究图谱与大字身份句。
- 放大现有鼠标视差，使图谱、景深层和坐标信息产生可感知但不眩晕的位移差。
- 标题分行揭示从简单淡入改为具有方向性的错位入场；首屏加载完成后出现一次克制的图谱脉冲。
- CTA 增加磁吸位移、边缘高光与按压反馈，不使用蓝紫渐变或发光按钮。
- 加入随滚动变化的章节进度标记，帮助访客理解页面是一段连续构建轨迹。

### 4.2 构建轨迹

在首屏与个人故事之间增加“问题 → 实验 → 系统 → 产品”四阶段轨迹。它不是四张卡片，而是一条横向/纵向响应式路径：

- 问题：科研流程依赖经验、难以复用。
- 实验：在真实分析中迭代脚本、Agent 和检查点。
- 系统：将阶段约束、Skills、Hooks、评审与共享方法库组合起来。
- 产品：形成 scrna-omics，并将方法迁移到其他产品。

桌面端随滚动逐段点亮节点、线条和证据数字；移动端改为纵向路径。所有内容在 JavaScript 关闭时仍可阅读。

### 4.3 投入证据

CC-Switch 本地数据库可核验的快照区间为 2026-06-07 至 2026-07-09：Claude Code 与 Codex 共记录 19,313 次工作流请求、8 个模型、约 5,370,092,638 Token。首页仅展示：

- `近 30 天`
- `8 个模型`
- `5B+ 上下文处理量`

并用说明明确：这是本地工具记录的工作流处理量，用来说明构建密度，不代表能力、用户规模或产品成果。当前不合并 Antigravity 与未纳入 CC-Switch 的其他 API，用量口径不能核验时不展示。

数字随对应轨迹进入视口后计数显现；`prefers-reduced-motion` 下直接显示最终值。

### 4.4 项目与微交互

- 项目标题、预览图与鼠标指针建立更明显的局部响应；预览采用景深、轻微倾斜和切换过渡。
- 展开项目时强化标题到正文的连续位移，避免突然出现大块内容。
- 小产品索引增加方向性 hover 与链接反馈，但保持列表而非卡片墙。
- 媒体视频区继续默认收起，展开时采用高度与透明度过渡。
- 不引入 3D 框架、大型依赖或强制自定义光标。

## 5. 报名填写稿

新增一份不进入公开页面的 Markdown 填写稿，按真实表单顺序覆盖全部可写字段：

### 5.1 基本信息

- 昵称与基本身份：沿用真实信息，不生成敏感字段。
- 擅长技能：从“其他”改为可辨识的组合表达，突出生命科学研究、生物信息学、AI 工作流与产品开发。
- 自我介绍：用 1000 字以内说明背景、核心项目、工作方式和希望寻找的协作者。
- 技术或能力：按科研、开发、AI、产品四条能力链描述，避免“高级分析”“服务器管理”等宽泛标签。

### 5.2 深入了解

- 最近兴奋的 Idea：以生命科学 AI4S 产品为题，讲清研究中的具体痛点、scrna-omics 的现有验证和仍待解决的问题。
- 其他活动/项目/奖项/实习：用结构化短条目列出代表项目、两次 AI+ 创新大赛、校级创新创业项目、内容与社群、GoGoWork 团队经历；不把在投论文写成已发表。
- 相关链接：个人网站、GitHub、scrna-omics 宣传站、小红书主页，以及最强的公开代表作；删除示例链接与无关平台占位。
- 为什么参加：回答 AdventureX 的现场共创特性为何与个人当前阶段匹配，并说明可为团队带来的科研场景、全栈交付和产品判断。
- 对活动的期待：聚焦认识能共同做 AI4S 产品的人、获得算力/平台/技术反馈，并在五天内交付可体验原型；不写成单向索取资源。

### 5.3 短答问题

三组题目均重新比较后选择最能形成互补的一题。当前设计默认保留：

1. 给 2036 年的自己写信：展示长期尺度、对行动与诚实的要求。
2. 被低估的微小技术趋势：讨论可验证的 Agent 工作流基础设施，而非泛泛预测“大模型将改变世界”。
3. 荒诞但有趣的无用发明：展示幽默、想象力和产品定义能力，避免再次回答 AI4S。

自由补充用一句轻松但真实的个人表达，不重复履历。

### 5.4 附件

- 简历使用现有两页 A4 中文 PDF，经事实修订后重新生成。
- 作品集使用当前主页的可发布版本；不把填写稿或本地统计原始数据库上传。
- 其他附件默认不上传，除非最终发现有一份能够显著补充作品集而非重复内容的材料。

## 6. 文案标准

- 所有中文正文使用 `humanizer-zh` 审阅。
- 使用第一人称和具体动词，优先写观察、判断、动作和结果。
- 避免“真正”“赋能”“不只是……更是……”“我没有离开……我只是……”等模板句。
- 激情通过持续投入、真实项目和下一步行动体现，不直接堆叠“热爱、梦想、坚定”。
- 三篇 SCI 仅写在投；科研立项明确校级；GoGoWork 明确团队身份；未来事项写计划或筹备。
- 每道题控制在表单限制内，并保留字数余量。

## 7. 实现边界

- 保持静态 HTML/CSS/JavaScript 架构，不迁移 React。
- 新动效以 CSS、IntersectionObserver 和 requestAnimationFrame 实现。
- 不引入外部跟踪、远程字体依赖或需要 API 密钥的运行时功能。
- 不自动填写或提交 AdventureX 表单；用户确认最终稿后再手动复制。
- 不公开原始 Token 日志、供应商、费用、Cookie、账号信息或私人会话内容。

## 8. 验证

- 1440 × 1000、1024 × 768、390 × 844 三种尺寸无横向溢出、遮挡或动效跳变。
- 检查首页首次入场、滚动轨迹、项目展开、磁吸按钮、视频展开和键盘焦点。
- `prefers-reduced-motion` 下移除视差、计数动画、平滑位移和循环脉冲。
- 页面在本地 HTTP 与 `file://` 模式均可使用核心内容。
- 报名稿逐题核对字段、字数、事实、链接与隐私边界。
- 简历重新生成后保持两页 A4、中文可选取，并通过事实约束测试。
