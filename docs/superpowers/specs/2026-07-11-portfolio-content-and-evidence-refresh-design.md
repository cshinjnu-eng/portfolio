# 个人主页内容与作品证据修订设计

## 目标

保留已确认的“会生长的研究图谱”视觉方向，修复四个具体问题：主页不再出现真人头像；中文文案不再带有明显的 AI 模板感；TimeBox 展示真正的暖纸色 v2 界面；小产品不再只有项目名和一句标签。

## 1. 人物视觉

- 网站删除 `assets/brand/portrait.jpg` 的所有引用。
- 故事章的照片改为一幅新生成的抽象人物剪影：侧身或背影，无可识别五官，有花粉、细胞网络和水墨扩散的形态。
- 色彩沿用米纸色、墨黑、朱红和植物绿，不增加蓝紫渐变、玻璃拟态或发光边缘。
- 剪影是个人故事的插图，不伪装成肖像照，替代文本使用“抽象人物剪影与生命科学网络”。

## 2. TimeBox 证据更新

- 真实界面来自 `/Users/chensihan/Desktop/📁开发项目/timebox/private` 的 `feat/v2-design` 工作树，不再使用旧的深色 Hermes 手机截图。
- 以 `src/styles/timebox-v2.css` 的暖纸色系统为判定依据：页面背景 `#faf6f0`，下沉背景 `#f1ebe1`，卡片为白色或 `#f7f2ea`。
- 本地运行当前分支，获取至少两张真实界面：一张表现时间/成长记录，一张表现 Hermes 或 AI 洞察。
- 如果登录门阻止获取内部界面，优先使用仓库内的开发数据或已有认证会话。不制造不存在的界面。
- 作品集中 TimeBox 的主视觉以真实截图为主，宣传落地页只做背景或辅助。

## 3. 文案修订

- 范围包括 `index.html`、`data/projects.js`、`resume/resume_data.py` 及简历页面中的所有中文可见文字。
- 使用 `humanizer-zh` 逐段检查：删掉宣传式大词、否定式金句、刻意的三段式和模糊归因；优先写具体的工作、结果和个人判断。
- 保留主线“跨医学、科研与产品的独立构建者”，但正文不反复复述这个标签。
- 保留事实边界：3 篇 SCI 为在投；项目为校级立项；GoGoWork 为团队项目；尚未发生的展出或宣传使用“计划”或“筹备中”。
- 简历语气比网站更克制；网站可以使用第一人称，但不写广告口号。

## 4. 小产品与研究工具

- 小产品区不使用统一的卡片阵列，改为可扫读的索引行：项目名、类型、一句具体说明、个人角色或产出、外部链接。
- 首批收录：
  - `scAIreport`：将 Seurat 聚类导出为模型可读的 JSON，用于细胞注释和结果复核。
  - `scRNA-seq-gdT-psoriasis`：银屑病外周血 γδT 细胞的完整单细胞分析，体现科研过程和可复核性。
  - `Paper4AI`：将论文网页整理成干净 Markdown，方便交给模型阅读。
  - `Ask LLM & Link`：在 Obsidian 中将选中内容提交给模型，再把回答保存为可链接笔记。
  - `dental-trauma-kids-guide`：儿童牙外伤科普互动页。
  - `anesthesia-checklist`：麻醉流程的结构化核对工具。
  - `image-slicer-web`：在浏览器中分割长图，适配常用内容平台。
  - `claude-adapter`：OpenAI 到 Anthropic Messages API 的请求格式转换层。
- 研究项目和工具型产品共用一个索引，但用类型和说明区分，不把科研仓库包装成 SaaS。

## 5. 验证与发布

- 在 1440px 桌面端和 390px 移动端检查剪影、TimeBox 截图和小产品索引，不出现水平溢出。
- 检查项目展开、视频懒加载、减少动效和键盘焦点。
- 重新生成简历 PDF，保持两页 A4，并运行事实约束测试。

- 不为小产品制造虚构指标或用户数。
- 不将 TimeBox 私有仓库、登录信息或未公开代码上传到作品集仓库。
- 简历仍保留用户已提供的真人照片；“网站不用头像”不扩大为“简历也删除照片”。
