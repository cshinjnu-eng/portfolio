# 内容视频与简历证据增量设计

## 目标

在不扩张主页主线的前提下，补充“本硕博三无”的视频内容产出，并根据用户确认更正两次 AI+ 创新大赛奖项。

## 证据口径

- 账号快照来自 `docs/insights/小红书账号只读审计-2026-07.md` 和同目录公开笔记台账。
- 使用 `agent-reach` / `xhs` 按精确标题复核了 4 条公开视频的账号、类型、标题、笔记 ID 和封面。
- 项目内不保存 Cookie、`xsec_token`、后台原始返回或仅自己可见的笔记。
- 账号数据使用 2026-07 快照：24 条公开笔记、4 条公开视频、139,821 累计曝光、3,939 点赞、680 收藏。
- 对外页面保留用户原有“近 5,000 收藏与点赞”口径，并显示快照月份，避免把动态数据写成永久值。

## 网站展示

- 位置：保持在现有“MEDIA / COMMUNITY”章节，放在账号介介与 Claude 风险自测之后，不新建主导航章节。
- 交互：使用原生 `<details>` / `<summary>`。默认收起，摘要显示“4 条公开视频 · 展开查看”，不为此增加 JavaScript 状态。
- 布局：桌面端两列，移动端单列。使用封面、文字层级与分割线分组，不使用多层嵌套卡片。
- 加载：仓库只保存压缩封面，不复制四个大视频文件。点击项目打开小红书公开笔记。
- 指标：不在单条视频上显示点赞、收藏等会快速变化的数据。账号总览可显示带日期的快照。
